# Link Mismatch Audit — 2026-04-22

Research-only pass. No files were edited. Scope was the curated hardcoded-link
surfaces listed in the brief: landing pages under `src/app/**`, curated
libraries in `src/lib/`, `llms.txt` routes, and `relatedEpisodes:` frontmatter
arrays across `content/blog/**/*.mdx` and `content/podcast/**/*.mdx`.

Findings concentrate in the `relatedEpisodes:` frontmatter arrays on blog
posts that name a specific guest. The pattern is the same every time: the
post's title is about guest X, but the array contains one (or zero) actual
guest-X episodes plus two or three generic "Rider Support" / solo Anthony
episodes that look like autogenerated filler. When these render on the
live blog post, users see a "RELATED PODCAST EPISODES" block that mostly
or entirely fails to deliver conversations with the guest named in the
article.

Nothing new was found on `/start-here`, `/research`, `/about`, `/about/press`,
`/coaching`, `/coaching/triathlon`, `/coaching/[location]`, `/community/*`,
`/you/[slug]`, `/plan/*`, `/topics/*`, or in `src/lib/personas.ts`,
`src/lib/topics.ts`, `src/lib/testimonials.ts`, `src/lib/training-plans.ts`,
`src/lib/tools/reports.ts`, `src/app/llms.txt/route.ts`, or
`src/app/llms-full.txt/route.ts`. Every hardcoded episode slug in those
locations resolves to an episode whose `guest:` frontmatter matches the
label. The two /start-here cards that were already hand-fixed have been
excluded as instructed.

## Method notes

- Read all hardcoded `href={...}` / `slug:` arrays in every `page.tsx`
  under `src/app/` plus every file in `src/lib/` with internal `/podcast/`
  or `/blog/` references.
- Extracted every `relatedEpisodes:` frontmatter array from
  `content/blog/**/*.mdx` (190 files) and cross-referenced each episode
  slug against that episode's `guest:` and `title:` fields.
- No `relatedEpisodes:` arrays were found on podcast MDX files.
- Inline `[label](/podcast/ep-...)` markdown links were spot-checked;
  all sampled cases matched the target episode's guest.
- External URLs (YouTube, Apple, Spotify), auto-generated recommendation
  blocks driven by `RelatedContent` / `RelatedPosts`, and admin dashboard
  placeholder data were excluded per scope.
